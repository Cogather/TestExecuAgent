# Record Scripts

将用例步骤转换为可执行的分步脚本，并按“先复用、失败再重录”的规则推进。

## 输入

开始前必须具备：

- `case_id`
- 起始 URL

## 输出

输出到 `./<case_id>/`：

- `<case_id>_step1.py`
- `<case_id>_step2.py`
- ...
- 步骤处理结果汇总（复用成功 / 复用失败后重录 / 直接录制）

## 流程

### 1. 逐步骤执行

对每一个步骤 `N`，按以下规则处理：

1. 如果 `step_reuse_flags[N] = true`：
   - 先检查是否存在可复用脚本。
   - 若存在，则先执行该复用脚本。
   - 若执行成功（退出码为 `0` 且无阻断错误），直接进入下一步骤。
   - 若执行失败（非 `0`、异常、关键动作未完成），立即重录该步骤并覆盖 `./<case_id>/<case_id>_stepN.py`。
   - 若标识为 `true` 但未找到可复用脚本，直接进入重录流程。

2. 如果 `step_reuse_flags[N] = false`：
   - 不尝试复用，直接录制该步骤并输出到目标路径。

### 2. 重录规则

当步骤需要重录时，运行如下命令：

```bash
python -m playwright codegen --ignore-https-errors [URL] -o "./<case_id>/<case_id>_stepN.py"
```

重录后要求：

- 让用户确认该步骤脚本是否满意。
- 不满意则继续重录当前步骤。
- 满意后再进入下一步骤。

### 4. 完成汇总

所有步骤处理完成后，输出：

- 每个步骤最终脚本路径s
- 每个步骤的处理方式（复用通过 / 复用失败后重录 / 直接录制）
- 未处理成功的步骤（如有）

## 约束

- 必须按步骤顺序处理，不跳步。
- 复用失败只重录当前步骤，不影响已通过步骤。
- 不修改其他 skill 的输入输出契约。
- 不负责脚本修复、执行验证和结果判定。
